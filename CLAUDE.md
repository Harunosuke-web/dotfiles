# dotfiles — 運用ルール

macOS環境の設定をGNU stowで管理するリポジトリ。**GitHub上で公開されている**。

## 設計原則（変更時も必ず維持する）

1. **XDG Base Directory準拠** — 設定は`~/.config`、データは`~/.local/share`、状態は`~/.local/state`、キャッシュは`~/.cache`へ。ホーム直下に新しいドットファイルを増やさない
2. **1ツール1パッケージ** — `packages/<tool>/`にstow構造で置く。ツールの追加・削除が互いに独立してできる形を守る
3. **パッケージ管理の役割分担** — システムツール・重い依存はHomebrew（`homebrew/Brewfile`）、言語ランタイムはmise、シェル専用の軽量ツール・プラグインはzinit
4. **セキュリティ** — 秘密鍵・トークンは絶対にコミットしない。`.ssh`はホワイトリスト方式gitignore（`config`と`*.pub`のみ許可）。鍵の実体は1Passwordエージェントが持ち、ディスクに置かない
5. **実名・サブアカウント名を書かない** — 公開リポジトリのため。サブアカウントは`github-sub`という汎用名で表記する（SSHホスト別名・鍵ファイル名とも）

## 運用ルール

- **1ヶ月ルール** — 新ツールはまずbrewで入れて試し、1ヶ月使い続けたらBrewfileに書く。Brewfileは「インストールログ」ではなく「厳選リスト」として保つ
- **コメント方針** — 採用しなかった代替手段のコメントアウトは残さない（Gitの履歴が覚えている）。コメントは「今動いている設定の説明」だけにする
- **健診** — `./bin/doctor.sh`で設定と実環境のズレ（壊れたリンク・実体のない設定・Brewfile差分）を検査できる。棚卸しはまずこれから
- **コミット** — Conventional Commits形式。type/scopeは英語、説明と本文は日本語

## 構成

```
bin/         ユーザーが直接叩くスクリプト（setup-links / update / doctor）
scripts/     セットアップ用の内部スクリプト（common.sh に共通変数）
packages/    stowパッケージ群（1ツール1ディレクトリ）
homebrew/    Brewfile
docs/        セットアップ手順・スクリプト解説
man/         自作スクリプトのmanページ
```

## Claudeへの注意

- `~/.zshenv`や`~/.config/*`はこのリポジトリへのシンボリックリンク。**編集は必ずリポジトリの実体側**で行う（ホーム側を直接編集しようとするとsymlinkエラーになる）
- `GITHUB_USER_NAME`は`scripts/common.sh`が唯一の定義場所。他で再定義しない
- CLAUDE.mdはグローバルgitignoreの対象外（通常どおりGitで追跡する）。プロジェクト側の`.claude/`はignoreされる
- パッケージの一括削除や構成の大規模変更は、所有者に確認してから
